# humane v0.9.4

## `TimeAgo` takes `at` by value -- the pointer moves inside

`TimeAgo`'s one and only argument changes from `at *time.Time` to
`at time.Time`. Every real consumer's timestamp is a real, never-nil value
(a file's mtime, a record's created-at) -- `lambada`'s own `timeAgo`
`FuncMap` closure existed purely to take `&t` before calling
`humane.TimeAgo(&t)`, an adapter with no actual optionality behind it.
`TimeAgo` now does that conversion itself:

```go
func TimeAgo(at time.Time, opts ...TimeOptions) string {
	return DistanceInTime(&at, time.Now(), opts...)
}
```

`DistanceInTime` is unchanged -- still `at *time.Time`, still nil-aware via
`TimeOptions.WhenNil` (see `docs/COMMENTS.md`'s `TimeOptions.WhenNil` entry).
It's the explicit, no-surprises core; a caller reaching for it by name is
choosing full control over what a missing timestamp means, not a
convenience default. A version of this change considered extending
"no pointer" to `DistanceInTime` too, treating a nil `at` as `relativeTo`
itself instead of a configurable fallback string -- reverted during review,
since that's exactly the kind of implicit behavior change a "no magic" core
function shouldn't get.

## Upgrading

Every call site passing a pointer to `TimeAgo` needs to drop the `&`:

```go
// old
humane.TimeAgo(&t)

// new
humane.TimeAgo(t)
```

If your wrapper only existed to take that address, delete it -- `TimeAgo`
can go straight into a `template.FuncMap` or similar by name:

```go
// old
"timeAgo": func(t time.Time) string {
    return humane.TimeAgo(&t)
},

// new
"timeAgo": humane.TimeAgo,
```

`lambada` is the only known Go consumer and needs this follow-up pass --
see its own `docs/COWORK.md` once that happens.
