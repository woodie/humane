# humane v0.9.5

## Migrated tests off Ginkgo/Gomega to spec+expect

No public API change -- `HumanSize`, `TimeAgo`, and `DistanceInTime` are
untouched. This is a test-suite and tooling change only: `size_test.go`/
`time_test.go` now use [`sclevine/spec`](https://github.com/sclevine/spec)
(replaced with [`github.com/woodie/spec`](https://github.com/woodie/spec))
for `describe`/`context`/`it` structure and
[`github.com/woodie/expect`](https://github.com/woodie/expect) for
assertions, in place of `onsi/ginkgo`/`onsi/gomega`. `go.mod` drops both
and their entire indirect-dependency tree.

## The lighter BDD stack is good to go

This closes out the loop this stack needed to prove itself: `spec`'s own
suite, `gorderly`'s own suite, `lambada`/`lambada-mta`'s full production
test suite, and now `humane`'s all run on `sclevine/spec` + `woodie/expect`
+ `gorderly` (`go test -v ./... | gorderly -fd`/`-fv`), with zero Ginkgo/
Gomega left anywhere in this account's own Go libraries. `ginkgo-fd`
itself is the one deliberate exception -- it's a Ginkgo output formatter,
so dogfooding real Ginkgo output there is correct, not a holdout.

Real advantages confirmed across four repos of actual use, not just
theory: no BDD-framework binary to install (`spec`/`expect` are plain
`go test`-native), a real nested tree from `go test -v`'s own output via
`gorderly` (no `--json-report` round trip), and a generics-based matcher
library (`expect`) with the same `Expect(...).To(Matcher)` ergonomics
Gomega users already know. Good to go for any future Go project in this
account -- no reason to reach for Ginkgo/Gomega going forward.
