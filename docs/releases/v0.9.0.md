# humane v0.9.0

## A full API rethink -- package functions, new defaults, nil-aware `TimeAgo`

Breaking change throughout. Through `v0.5.0` this package mirrored
Foundation's own formatter-type API shape. With three real consumers now
shipped, the bar changed: feel as simple to drop into a Go HTML template as
ActionView's own helpers, while staying consistent with `humane-ruby` and
`humane-swift`. See `docs/COWORK.md`'s `v0.9.0` entry for the full
rationale. Versioned `0.9.0` (a jump from `v0.5.0`, matching the identical
jump `humane-ruby` and `humane-swift` make in the same session) rather than
`v0.6.0` or `v1.0.0` -- this redesign is the most considered this API has
ever been, informed by three real consumers rather than a first guess, but
`0.9.0` treats it as a release-candidate for `v1.0.0` rather than a full
stability promise just yet.

```go
humane.SizeFormatter{}.Format(225935)              // old
humane.HumanSize(225935)                           // new -- "226 KB"

humane.NewTimeFormatter().Format(at, now)          // old
humane.TimeAgo(&at, now)                           // new -- "about 15 hours ago"
```

- **No more instantiation.** `SizeFormatter`/`TimeFormatter` types and
  `NewTimeFormatter()` are gone, replaced by package-level `HumanSize`/
  `TimeAgo` functions. Configuration is a trailing, optional
  `TimeOptions` argument instead of struct fields set ahead of a call.
- **`HumanSize` rounds to 3 significant figures**, not 2 -- fixes
  `5,240,000,000` -> `"5.24 GB"` (previously `"5.2 GB"`), and bakes in two
  other real-hardware findings: `0` -> `"Zero KB"`, and byte-scale values
  spell out `"bytes"` (`"7 bytes"`) rather than using a `"B"` label. See
  `docs/COMMENTS.md` for the derivation.
- **`TimeAgo`'s `Approximate` now defaults to `true`**, matching
  ActionView's own always-on-past-the-hour-boundary behavior, and matching
  what every real consumer already passed explicitly. `IncludeSeconds`
  stays `false` by default, unchanged. `TimeOptions.Approximate` is a
  `*bool`, not a `bool` -- see the README's "Why `Approximate` is a
  pointer" section for why a plain `bool` would silently reintroduce a
  zero-value bug this package already fixed once for `IncludeSeconds`.
- **`TimeAgo` takes `at *time.Time`** and a new `TimeOptions.WhenNil`
  string, returned as-is (no formatting) when `at` is `nil` -- lets a
  caller collapse a guard-then-fallback dance into one call.

## Upgrading

Every call site needs a change:

```go
// old
f := humane.NewTimeFormatter()
f.Format(t, now)

// new
humane.TimeAgo(&t, now)
```

If you were relying on `Approximate: false` (the old default) for exact
wording rather than "about" phrasing, pass it explicitly now:

```go
humane.TimeAgo(&t, now, humane.TimeOptions{Approximate: humane.Bool(false)})
```

`lambada` is the only known Go consumer and needs this follow-up pass --
see its own `docs/COWORK.md` once that happens.
