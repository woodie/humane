# humane v0.9.3

## `DistanceInTime`/`TimeAgo` split -- ActionView's naming pair, not just its defaults

Still converging on the final API -- versioned `0.9.3`, not `1.0.0`, on
purpose. The old two-argument `TimeAgo` is renamed `DistanceInTime`; a new
one-argument `TimeAgo` takes its place. Same naming split as ActionView's
own `distance_of_time_in_words` (explicit two-input core) and
`time_ago_in_words` (one-input convenience wrapping it with `Time.now`) --
deliberately borrowed, not invented. See `docs/COWORK.md`'s `v0.9.3` entry
for the full rationale, including the naming-family policy this establishes
(short names like `TimeAgo`/`HumanSize` stay reserved for "format a string
for a view"; anything returning something richer later gets a longer name
instead of competing for the short one).

```go
now := time.Now(); mtime := now.Add(-180 * time.Second)

humane.TimeAgo(&mtime, now)   // old -- "3 minutes ago"

humane.DistanceInTime(&mtime, now) // new -- explicit relativeTo, same output
humane.TimeAgo(&mtime)             // new -- one argument, relativeTo is time.Now()
```

- **`DistanceInTime`** is the explicit, fully-tested core -- unchanged
  behavior from the old `TimeAgo`, just renamed. This is what tests should
  call.
- **`TimeAgo`** is a new one-argument convenience for the common "drop into
  a view" case. Supplies `time.Now()` as `relativeTo` internally; every
  option (`Approximate`, `IncludeSeconds`, `WhenNil`) still applies.
- Not invented from scratch: `lambada`'s own `listingTemplate` FuncMap was
  already hand-wrapping the old two-argument `TimeAgo` in a closure just to
  supply the clock, after previously trying and reverting an
  explicit-`now`-in-the-template design for the same reason. `zouk`'s
  `ScanEntry.timeAgo` computed property did the equivalent by hand in
  Swift. The new one-argument `TimeAgo` here (and `humane-ruby`/
  `humane-swift`'s matching convenience) lets both drop their wrapper.

`HumanSize` is unchanged in this release.

## Upgrading

Every call site using the old two-argument `TimeAgo` needs a rename, not a
behavior change:

```go
// old
humane.TimeAgo(&t, now)

// new
humane.DistanceInTime(&t, now)
```

If the call site was only ever supplying `time.Now()` for `relativeTo` (the
common case in a template FuncMap), simplify to the new one-argument form
instead:

```go
// old
"timeAgo": func(t time.Time) string {
    return humane.TimeAgo(&t, time.Now())
},

// new
"timeAgo": func(t time.Time) string {
    return humane.TimeAgo(&t)
},
```

`lambada` is the only known Go consumer and needs this follow-up pass --
see its own `docs/COWORK.md` once that happens.
